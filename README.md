# Путёвка в жизнь

**«Путёвка в жизнь»** — это социальный проект, направленный на поддержку абитуриентов, оказавшихся в сложной жизненной ситуации. Система помогает участникам на всех этапах поступления и адаптации в учебных заведениях.

## 🎯 Основные задачи

- 🧑‍💻 **Личный кабинет благополучателя**  
  Удобный интерфейс для участников программы, где они могут отслеживать свой путь, получать задания, общаться с кураторами.

- 📋 **Упрощение процесса сбора данных**  
  Простые формы и автоматизация обработки анкет, отчётов и другой информации.

- 📈 **Трекинг прогресса и успехов**  
  Система отслеживания достижений, целей и образовательного прогресса.

## 🛠 Технологии

- Django  
- PostgreSQL  
- Docker  
- Bootstrap

## Beta Deploy

В репозитории есть GitHub Actions workflow для автодеплоя на beta-сервер.

Он запускается:

- при `push` в любую ветку, имя которой оканчивается на `--beta`
- вручную через `workflow_dispatch`, если выбранная ветка тоже оканчивается на `--beta`

Для работы workflow нужно завести GitHub Secrets:

- `BETA_DEPLOY_HOST` — адрес сервера
- `BETA_DEPLOY_USER` — SSH-пользователь на сервере
- `BETA_DEPLOY_SSH_KEY` — приватный SSH-ключ для входа на сервер
- `BETA_DEPLOY_PATH` — директория проекта на сервере
- `BETA_DEPLOY_ENV_FILE` — содержимое `.env` для beta-окружения, опционально

Workflow синхронизирует репозиторий на сервер по SSH через `rsync`, затем выполняет `docker compose up -d --build --remove-orphans` в директории деплоя.

Важно: workflow использует одну директорию `BETA_DEPLOY_PATH`, поэтому серверное beta-окружение всегда отражает последнюю beta-ветку, которая была задеплоена.
